## P3: 开发者体验快速修复

### 1. request.js 超时优化

**问题**: `uni.request()` 未设置 timeout，默认 10 秒，本地开发时后端不可用等待时间过长。

**方案**: 设置 `timeout: 5000`（5 秒），足够覆盖本地 API 调用。

**变更**: `frontend-mini/src/api/request.js` — 新增 `timeout` 参数

### 2. manifest.json 填入测试 AppID

**问题**: `appid` 为空字符串，导致 `uni.login()` 在微信开发者工具中可能失败。

**方案**: 填入测试 AppID `wxffc042f0b83d2419`，使微信 SDK 调用不会因空 AppID 报错。

**变更**: `frontend-mini/src/manifest.json` — `mp-weixin.appid`

### 3. uni.login() 错误降级

**问题**: `store/user.js` 中 `uni.login()` 无 try-catch，微信开发者工具中可能因环境差异失败且无降级。

**方案**: 包裹 try-catch，失败时 console.warn 并 fallback 到 `mockLogin()`。

**变更**: `frontend-mini/src/store/user.js` — `login()` 方法

### 数据流

```
用户登录 (role: admin)
  ↓
userInfo.role = "admin" (从 JWT payload 解析)
  ↓
me.vue: v-if="userRole === 'admin'" 显示管理端入口
  ↓
点击进入 admin-dashboard
  ↓
调用 GET /api/v1/admin/dashboard/overview
  ↓
显示统计卡片 + 快捷操作
  ↓
点击"处理订单" → admin-orders
  ↓
调用 GET /api/v1/admin/orders?status=pending
  ↓
待确认订单显示 [确认] [拒绝] 按钮
  ↓
点击确认 → POST /api/v1/admin/orders/{id}/confirm
点击拒绝 → uni.showModal(editable) → POST /api/v1/admin/orders/{id}/reject
```

### 权限控制

- **前端**: `me.vue` 中 `userRole` 计算属性控制入口显示
- **后端**: `RoleInterceptor` 拦截 `/api/v1/admin/**` 路径，非 admin 角色返回 403
- **Dashboard/Orders 页面**: API 调用失败时（403）自动跳转返回
