# 租车应用 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 背景
个人自营租车业务，面向旅游场景用户提供车辆出租服务。项目从小规模起步，先跑通"展示-预订-确认"核心流程。

### 1.2 目标用户
- **租客**: 旅游出行需要自驾的用户，通过微信小程序访问
- **管理员**: 车主本人，通过电脑端管理车辆和订单 + 小程序简化版管理端

### 1.3 业务模式
自营租车，车辆归车主所有，用户到店自取，按天计费。

### 1.4 产品定位
轻量级租车工具，MVP 阶段跑通核心流程，架构设计预留后期扩展能力。

## 3. 用户故事

### 3.1 小程序用户端

#### US-01: 微信一键登录
> 作为一名租客，我希望用微信一键授权手机号快速注册和登录，以便我能够使用租车服务。

**验收标准**:
- [ ] 点击"微信登录"按钮，弹出手机号授权弹窗
- [ ] 授权成功后自动创建账号并登录
- [ ] 登录态保持 7 天（微信 session 自动续期）
- [ ] 后端记录用户 wechat_openid 和 phone 的绑定关系

#### US-02: 浏览车辆
> 作为一名租客，我希望查看所有可租车辆的信息，以便我选择合适的车型。

**验收标准**:
- [ ] 按列表展示所有上架车辆
- [ ] 显示车型名称、封面图、日租金（区分工作日/周末价）
- [ ] 支持按价格区间筛选
- [ ] 已租完/下架车辆不展示

#### US-03: 查看车辆详情
> 作为一名租客，我希望查看车辆的详细信息，以便我了解车辆的具体情况。

**验收标准**:
- [ ] 展示车辆多图、品牌型号、座位数、变速箱类型
- [ ] 展示车辆描述/使用规则
- [ ] 展示不同日期类型的价格（工作日/周末/节假日）
- [ ] 底部有"立即预订"按钮

#### US-04: 预订下单
> 作为一名租客，我希望选择租期并下单，以便我预订车辆。

**验收标准**:
- [ ] 选择取车日期和还车日期
- [ ] 自动计算租期天数
- [ ] 根据日期类型（工作日/周末/节假日）自动计算总价
- [ ] 展示价格明细（每天的价格构成）
- [ ] 下单前需勾选同意用户协议
- [ ] 提交订单后订单状态为"待确认"
- [ ] 同一辆车在同一时间段不能被重复预订
- [ ] （v1.5 支付接入后）提交订单后跳转微信支付

#### US-05: 查看我的订单
> 作为一名租客，我希望查看我的订单状态，以便我了解预订进度。

**验收标准**:
- [ ] 按时间倒序展示订单列表
- [ ] 显示订单状态：待确认、已确认、进行中、已完成、已取消、已拒绝
- [ ] 点击进入订单详情
- [ ] 待确认状态的订单可以取消

#### US-06: 用户协议
> 作为一名租客，我希望在下单前阅读用户协议，以便我了解租车规则和责任。

**验收标准**:
- [ ] 下单页面展示协议勾选框
- [ ] 点击可展开查看协议全文
- [ ] 未勾选无法提交订单

#### US-07: 订阅消息通知
> 作为一名租客，我希望在订单状态变更时收到通知，以便我及时了解订单进展。

**验收标准**:
- [ ] 下单时引导用户授权订阅消息
- [ ] 管理员确认订单后推送通知
- [ ] 管理员拒绝订单后推送通知（含拒绝原因）

### 3.2 PC 管理端

#### US-08: 仪表盘
> 作为一名管理员，我希望看到营收概况，以便我快速了解业务状况。

**验收标准**:
- [ ] 今日订单数、今日收入
- [ ] 本月订单数、本月收入
- [ ] 当前进行中订单数
- [ ] 当前可租车辆数

#### US-09: 车辆管理
> 作为一名管理员，我希望添加和管理我的车辆信息，以便用户能看到并预订。

**验收标准**:
- [ ] 列表展示所有车辆（含下架）
- [ ] 新增车辆：填写车型信息、上传照片、设置价格
- [ ] 编辑车辆信息
- [ ] 上架/下架操作
- [ ] 删除车辆（软删除）

#### US-10: 订单管理
> 作为一名管理员，我希望处理用户订单，以便确认或拒绝租车请求。

**验收标准**:
- [ ] 列表展示所有订单，按状态筛选
- [ ] 查看订单详情（用户信息、车辆、租期、价格）
- [ ] 待确认订单可以"确认"或"拒绝"
- [ ] 确认订单后通知用户
- [ ] 进行中的订单可以标记"完成"

#### US-11: 价格设置
> 作为一名管理员，我希望配置周末/节假日价格，以便在不同时段灵活定价。

**验收标准**:
- [ ] 为每辆车设置工作日价格和周末价格
- [ ] 可配置特定日期为节假日并设置独立价格
- [ ] 支持批量设置日期类型（如设置某个日期区间为节假日）

### 3.3 小程序管理端（简化版）

#### US-12: 快速查看订单
> 作为一名管理员，我希望在手机上快速查看订单列表，以便我外出时也能了解业务状况。

**验收标准**:
- [ ] 按时间倒序展示订单列表
- [ ] 按状态筛选（待确认、进行中）
- [ ] 显示关键信息：用户手机号后4位、车型、租期、价格

#### US-13: 快速处理订单
> 作为一名管理员，我希望在手机上确认或拒绝订单，以便及时处理用户请求。

**验收标准**:
- [ ] 待确认订单可以"确认"或"拒绝"
- [ ] 拒绝时需填写原因（可选填，但建议填）
- [ ] 确认后自动通知用户

#### US-14: 订阅消息
> 作为一名管理员，我希望有新订单时收到推送通知，以便我及时处理。

**验收标准**:
- [ ] 用户提交订单后推送新订单通知
- [ ] 点击通知跳转到小程序管理端订单详情页

## 4. 数据模型（概要）

### 4.1 核心实体

```
User (用户)
├── id
├── phone              ← 微信授权获取
├── wechat_openid      ← 新增，微信唯一标识
├── nickname
├── role               ← 新增，user / admin
├── created_at
└── status

Vehicle (车辆)
├── id
├── name (车型名称)
├── brand
├── seats (座位数)
├── transmission (变速箱: auto/manual)
├── description
├── images (图片URL数组)
├── weekday_price (工作日价格)
├── weekend_price (周末价格)
├── holiday_price (节假日价格，可选)
├── status (上架/下架)
└── created_at

Order (订单)
├── id
├── user_id
├── vehicle_id
├── start_date (取车日期)
├── end_date (还车日期)
├── total_price
├── status (待确认/已确认/进行中/已完成/已取消/已拒绝)
├── payment_status (未支付/已支付/已退款) ← v1.0 固定为 unpaid
├── payment_id (微信支付流水号) ← v1.5 填充
└── created_at

HolidayConfig (节假日配置)
├── id
├── name (节假日名称)
├── start_date
├── end_date
├── price_multiplier (价格倍率) 或 fixed_price
└── year

UserAgreement (用户协议)
├── id
├── content
├── version
├── is_active
└── updated_at

MessageSubscription (消息订阅记录) ← 新增
├── id
├── user_id
├── template_id         ← 订阅消息模板ID
├── status              ← accepted / revoked
└── created_at
```

## 6. 业务流程

### 6.1 预订流程（v1.0 无支付）

```
用户浏览车辆 → 选择租期 → 查看价格明细 → 勾选用户协议 → 提交订单
  → 订单状态: 待确认
  → 管理员确认 → 订单状态: 已确认 → 推送订阅消息通知用户 → 到店取车
  → 管理员标记进行中 → 订单状态: 进行中
  → 还车 → 管理员标记完成 → 订单状态: 已完成
```

### 6.2 拒绝流程

```
订单待确认 → 管理员拒绝 → 订单状态: 已拒绝
  → 推送订阅消息通知用户
  → （v1.5 支付接入后）触发自动退款
```

### 6.3 取消流程

```
用户取消(待确认状态) → 订单状态: 已取消
  → （v1.5 支付接入后）触发自动退款
```

### 6.4 用户登录流程

```
用户打开小程序 → 点击登录
  → 调用 wx.getUserPhoneNumber 获取手机号 code
  → 后端用 code 换取手机号 + 同时获取 wechat_openid
  → 首次：创建用户并绑定 phone + openid
  → 非首次：通过 openid 查找用户，更新 session
  → 返回 JWT token
```

|------|------|

## 9. 终端变更影响说明

### 从 H5 改为微信小程序的关键变化

| 变化点 | H5 方案 | 小程序方案 |
|--------|---------|-----------|
| 登录方式 | 手机号+短信验证码 | 微信一键获取手机号 |
| 支付 | JSAPI 支付 | 小程序支付（v1.5） |
| 图片展示 | `<img>` 标签 | `<image>` 组件 |
| 通知 | 无 | 订阅消息推送 |
| 前端框架 | React SPA | 微信原生小程序 / uni-app / Taro |
| 部署方式 | Nginx 静态文件 | 微信小程序审核发布 |
| 管理端 | PC 网页 | PC 网页 + 小程序管理端（同一小程序内通过角色区分） |

### 同一小程序内区分用户端和管理端

- 通过用户 `role` 字段（user / admin）控制
- 普通用户：首页为车辆列表
- 管理员：首页为用户端，底部 Tab 增加"管理"入口
- 同一套代码，通过路由和角色权限区分
